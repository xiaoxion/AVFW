    1. What are some of the ways that you could change your HTML on your Demo app to be more accessible? (Remember, you can use the Section tag with jQuery, for instance)

    The simplest and quickest method that I can think of would be to support screen readers. This would enable my site to be navigable despite not being built for those users. I could also provide an option to have larger interfaces so the user would be able to have a smaller margin of error when he clicks.

    2. Visit a video service on the web (like YouTube) and search for "Screen Reader Demo" and "Mobile Screen Reader" - what are some of the common frustrations (or common support mechanisms) that you see among the examples that are given - how could you fix them (or use them)?

    Some of the more common frustrations is that the common HMTL markup is not entirely appropriate for screen reading.

    For example, the user will have a line of Bold or Italic text, and since it is semantically marked-up as Bold or Italic, when the screen reader reads that section, it does not recognize the bold or italicized text as being different from other text. Thus the page is stoic and the user is never introduced to some of the emphasized text and content.

    In order to aide this, I would look to not use the basic Bold and Italic, but rather use the Strong and Emphasis tags to make sure our content is read correctly by the reader and the user is exposed to the emphasized content as we intended it to be.

    3. Devices: Please list two devices (and some of their features) that are made specifically to support Accessibility.

    I believe that touch screen devices and smartphones both have an amazing amount of potential to be accessible for almost all types of users.

    Firstly, cell phones have a built in accessibility service call TTY mode, which enables blind users to text as it will automatically convert the messages to voice so that they are able to listen to them. It is also paired with talk back, a service that is much like a cell display reader, but since it is much more fine-tuned for the specific OS and increases the usability for blind users immensely as it reads almost all of the important information for the user and lets them access their device by means that would normally be inaccessible.

    The other means is touchscreen devices. They provide a very simplified means of communication for deaf individuals. The beauty of touchscreen is that because the interface is entirely visual, they can interact with the device and other users with text and visual cues. They do not need to talk nor be able to hear to use them. This opens communication for many people. Also, as a side note, the technology is evolving to the point that the touchscreen will be able to be “felt”. Meaning that even if a user is not able to see, hear, or talk, so long as they can feel, they can interact with the device. Which the premise looks amazing.

    4. Mobile Apps: Please list two apps (either Android or iOS, or both) that are made specifically to support Accessibility.

    Audiobooks is a great app that is built specifically to support accessibility. Since the nature of the site and app is to provide and audio version of written text, it is naturally blind friendly, but they added more to the interface, such as spoken prompts when a user interacts with the display, that make it entirely usable for someone that is visually impaired or blind.

    The other is Eyes-Free Shell. This interface is actually not visual interface at all. It is built entirely to be used without visual cues, and the large letters make it not only accessible, but built entirely for blind users. It aids a lot as it is not only a home replacement but comes with many other sub-apps that lets the user interact with the phone that would otherwise be way too difficult.
